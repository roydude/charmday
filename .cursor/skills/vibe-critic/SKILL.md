## Workflow
1. Read the full idea/spec/flow.
2. Run friction + shareability checks.
3. Output specific fixes (copy + flow), not vague advice.

## Checks
- **Friction**: If it takes >30 seconds, fail it and cut steps.
- **Shareability**: If it is not Story-worthy (funny, shocking, ego-boosting), fail it and upgrade the payoff.
- **Clarity**: If result types blur together, rewrite labels so they are instantly distinct.

## Output
- Verdict: Pass / Fail (+ one-line reason)
- Blockers: what must change before implementation
- Suggestions: concrete rewrites for hook/result/CTAs + the shortest improved flow
