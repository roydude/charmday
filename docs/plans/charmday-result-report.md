# Charmday 기획안 — 결과 리포트(Result Report) 강화 v0.1

## Vibe Critic 리뷰 (Devil’s Advocate)
**Verdict: Pass (if this doc is followed)** — “좋은 방향”이고, 아래 보강(카피 뱅크 + MVP 스코프 + 앵커 네이밍)을 적용하면 밋밋해질 리스크를 크게 줄일 수 있다.

### Blockers (구현 전에 반드시 보강)
1) **카피 뱅크 부재**: `absurd_specific_line`, Buff/Debuff, Don’t 라인이 없어서 “맞춘 느낌/밈성”이 약해질 위험
2) **MVP 스코프 불명확**: 모듈이 많아 10–20초 리포트 목표를 초과하기 쉬움(스크롤 피로)
3) **결과 라벨/앵커가 평범**: 오행 자체는 구분되지만 “저장하고 싶은 네이밍”이 부족

### 즉시 적용 수정안 (가장 짧은 개선 플로우)
Start → 입력 → `오늘의 부적템 받기` → (5s 로딩) → **Result Card(Top)** + `친구 소환` → **MVP Report(Why 3줄 + Do/Don’t + Lucky Item 1줄)** → 상품 CTA

---

## 목적
- 유저가 “왜 이 결과가 나왔는지”를 10–20초 안에 납득하게 만들고
- 스크린샷/공유를 자연스럽게 유도하며
- 쿠팡 클릭(구매 의도)을 **서사로** 연결한다.

## 한 줄 요약(북극성)
**결과는 2층 구조로 설계한다:** `Result Card(1초 공유용)` → `Result Report(10–20초 설득/전환용)`

---

## 전제(현재 구현 기준)
- 3-뷰 구조: 시작(`view-start`) → 로딩(`view-loading`, 5초) → 결과(`view-result`)
- 입력: 연/월 필수, 일/시간 미상 옵션
- 분석 산출: `lacking(부족 오행)`, `dominant(우세 오행)`, `yinyang`, `scores`, `mode`, `confidence`, `mainItem/lightItem/strongItem/extras`
- 메인 CTA: `가볍게 액땜하기` / `확실히 보정하기` / `쿠팡에서 바로 보기`

관련 파일
- `index.html` (결과 섹션 마크업)
- `app.js` (`analyze()`, `renderResult()`)
- `styles.css` (결과 카드 스타일)

---

## 문제 정의(현재 결과 리포트의 “아쉬움”)
1) **왜 이 결과인지**가 1~2문장으로 끝나서 “나한테 맞춘 느낌”이 약함  
2) 상품 추천이 “좋아 보이긴 하는데” **서사적 이유(상황/행동/금기/버프)**가 부족함  
3) 공유/재방문을 만드는 **짧은 문장(밈/한줄저격)**과 “복붙 캡션”이 없음

---

## MVP 스코프(10–20초 리포트 보장)
모듈을 “필수 3개 + 옵션 2개”로 쪼개서, **읽기 시간**을 통제한다.

### 필수(MVP) — 무조건 넣기
1) **Result Card** (headline/subhead/absurd line + share)
2) **Why 3 Signals** (3 bullets)
3) **Do / Don’t + Lucky Item** (각 1줄, 총 3줄)

### 옵션(v1+) — 성과 나오면 추가
- Buff/Debuff (각 1줄)
- Gift Angle (1줄)

---

## 결과 2층 구조 스펙

### A. Result Card (Share Bait, 1초)
목표: **아무 맥락 없이도** 이해되고 저장/공유가 되는 카드.

카드 구성(고정)
- `headline`: 결과 제목(짧고 라벨 가능) — 예: “오늘은 수(水) 보강”
- `subhead`: 분위기 1줄 — 예: “회복이 먼저인 날, 과열은 꺼줘”
- `absurd_specific_line`: “이상하게 정확한 한 줄” — 예: “요즘 너, 답장 느린 거 티남”
- `anchor`: 결과별 상징 1개(아이콘/오브젝트/컬러)
- `micro-cta`: 4–8자 — 예: “친구 소환”, “공유하기”

카드 카피 길이 가이드(ko)
- headline: 10–16자
- subhead: 18–30자
- absurd_specific_line: 14–26자

### B. Result Report (Conversion Bait, 10–20초)
목표: “왜 나야?” → “오늘 뭐 하지?” → “그래서 뭘 사?”를 한 화면씩 넘기듯 읽히게.

권장 모듈(각 1스크린 이하)
- **필수(MVP)**:
  1) **3 Signals (왜 이 결과인지)**: 근거 3개 bullet
  2) **Do / Don’t (행동 지시)**: 각 1줄
  3) **Lucky Item (서사 소품)**: 1줄(“상품명”보다 “변화”를 말하기)
- **옵션(v1+)**:
  - **Buff / Debuff**: 장점 1줄 + 주의 1줄
  - **Gift Angle**: “이 결과인 친구에게 선물하면 딱” 1줄(공유 + 구매 동시 유도)

읽기 시간 가드레일(ko)
- MVP 리포트 첫 화면: **최대 6줄**(bullet 3 + 1줄 + 1줄 + 1줄)
- 한 줄 길이: 18–30자 권장(문단 금지)

안전 가이드
- 의료/법률 단정 금지, 체형/외모 조롱 금지, 죄책감 유발(“안 사면 망함”) 금지
- 톤은 “가볍게 찔러주되, 상처는 내지 않게”

## Charmday용 데이터 매핑(현재 로직 그대로 활용)

### Result Card용 필드(생성 규칙)
- headline: `I18N[currentLang].summaryTemplate` 기반(현재 존재) + 더 짧게 압축한 별도 템플릿 권장
- subhead: `detailTemplate`를 더 “밈”스럽게 재가공(긴 문장 금지)
- absurd_specific_line: `lacking`, `dominant`, `yinyang` 조합으로 5~10개 룰 기반 랜덤(고정 문장 풀)
- anchor: `lacking` 오행에 1:1 매핑(색/아이콘)

## 결과 앵커(저장 욕구를 만드는 네이밍)
오행(wood/fire/earth/metal/water) 위에 “별명” 레이어를 붙여서 결과가 더 밈처럼 보이게 한다.

권장(ko) — `lacking` 기준
- wood(목): **새싹 모드** / anchor: 🌿 / 키워드: “시작·확장·루틴”
- fire(화): **점화 모드** / anchor: 🔥 / 키워드: “표현·연락·활력”
- earth(토): **고정 모드** / anchor: 🧱 / 키워드: “정리·지속·안정”
- metal(금): **컷팅 모드** / anchor: ✂️ / 키워드: “결단·경계·정렬”
- water(수): **회복 모드** / anchor: 💧 / 키워드: “휴식·유연·컨디션”

표기 규칙(예시)
- headline: “오늘은 **회복 모드(수)**”
- chip/바 영역은 기존 오행 라벨 유지(정보성), 카드 헤드는 별명으로 (밈성)

### Result Report용 필드(생성 규칙)
- 3 Signals:
  - signal1: `scores`에서 최저/최고 대비(“{lack}가 가장 낮게 찍힘” 식)
  - signal2: `yinyang` + `yinyangGap` 기반(“{yinyang} 쏠림이라 {tip} 필요”)
  - signal3: `mode` 기반 신뢰도 문구(“시간 포함/미상 보정”)
- Buff/Debuff:
  - buff: `dominant`의 장점 1줄(오행별 고정 카피)
  - debuff: `lacking` 상태의 오늘 흔한 실수 1줄(오행별 고정 카피)
- Do/Don’t:
  - do: 현재 `ELEMENT_TRAITS[lacking].action`를 확장(1개 → 2~3개 풀)
  - don’t: 오행별 금기 1줄(“불필요한 과열/충동구매/연락폭주” 같은 톤)
- Lucky Item:
  - `mainItem`을 “행동/감정 변화” 문장으로 연결(상품명 나열만 하지 말기)
- Gift Angle(옵션):
  - `dominant`/`lacking` 조합으로 “선물하면 좋은 대상”을 한 줄로(예: “요즘 과열된 친구한테는 수(水)템”)

## 카피 뱅크(최소) — 구현자가 바로 쓰는 문장 풀
아래 문장은 “룰 기반 랜덤”으로 섞어도 되고, 오행별로 3~5개씩만 먼저 넣어도 MVP가 산다.

### Buff / Debuff (오행별 3개씩)
- wood(목)
  - Buff: “시작만 하면 속도가 붙는 날.”
  - Buff: “새 루틴이 생각보다 쉽게 자리 잡음.”
  - Buff: “작은 진전이 크게 보상되는 타입.”
  - Debuff: “완벽주의로 첫 발을 못 떼기.”
  - Debuff: “남의 페이스에 휘둘려 기운 새기.”
  - Debuff: “계획만 늘리고 실행을 미루기.”
- fire(화)
  - Buff: “표현하면 관계/기회가 열린다.”
  - Buff: “주목받을수록 운이 붙는 흐름.”
  - Buff: “연락/결정이 빨라질수록 이득.”
  - Debuff: “흥분 상태에서 말이 앞서기.”
  - Debuff: “충동 결제/즉흥 약속 늘리기.”
  - Debuff: “과열로 컨디션 급락.”
- earth(토)
  - Buff: “정리하면 마음이 바로 안정된다.”
  - Buff: “루틴이 복리처럼 쌓이는 날.”
  - Buff: “작은 정돈이 큰 운을 만든다.”
  - Debuff: “불안해서 더 느려지는 패턴.”
  - Debuff: “완전히 준비될 때까지 시작 안 하기.”
  - Debuff: “걱정으로 현재 리듬 깨기.”
- metal(금)
  - Buff: “정리/결단이 곧 속도.”
  - Buff: “경계선을 잡으면 선택이 쉬워진다.”
  - Buff: “불필요한 것을 덜수록 운이 남는다.”
  - Debuff: “디테일 집착으로 완료를 못 내기.”
  - Debuff: “상대 반응을 테스트하다 피곤해지기.”
  - Debuff: “한 번에 다 끊으려다 폭발하기.”
- water(수)
  - Buff: “회복이 곧 실력, 부드럽게 가면 빨라진다.”
  - Buff: “컨디션만 올리면 판단력이 살아난다.”
  - Buff: “유연하게 우회하면 오히려 직진이 된다.”
  - Debuff: “피곤한데 억지로 버티기.”
  - Debuff: “기분 다운을 쇼핑/자극으로 메우기.”
  - Debuff: “약속 과다로 회복 시간 증발.”

### Absurd specific line (오행별 5개)
- wood(목)
  - “요즘 너, 시작 버튼만 누르고 멈춤.”
  - “할 일은 많은데 ‘첫 줄’이 제일 어려운 날.”
  - “계획은 완벽한데 실행이 1cm 부족.”
  - “새로 열어야 할 탭이 마음에 더 많음.”
  - “다시 ‘처음’으로 돌아가면 빨라져.”
- fire(화)
  - “갑자기 말 많아지는 날… 너도 알고 있지?”
  - “연락하고 싶은 사람, 지금 1명 떠오름.”
  - “감정이 먼저 달리고 논리는 뒤늦게 옴.”
  - “오늘은 ‘너무 솔직’해서 이득/손해가 갈림.”
  - “불필요한 과열만 꺼주면 완전 승.”
- earth(토)
  - “마음이 흔들리면 책상부터 어질러짐.”
  - “오늘은 안정이 최고 스펙.”
  - “계속 생각만 하면 피곤해지는 패턴.”
  - “너, 쉬는 것도 루틴이 필요함.”
  - “작게 정리하면 크게 안 무너져.”
- metal(금)
  - “선 긋는 말 한 번이 필요한 날.”
  - “애매하게 두면 계속 신경 쓰임.”
  - “오늘은 ‘정리’가 곧 운빨.”
  - “결정 미루면 손해 보는 타입, 오늘 특히.”
  - “깔끔하게 끊으면 속도가 붙어.”
- water(수)
  - “컨디션이 답을 알고 있는데 너만 무시함.”
  - “오늘은 회복이 실력.”
  - “무리하면 티 나게 망가지는 날.”
  - “감정이 마르는 느낌? 그거 맞아.”
  - “부드럽게 가면 오히려 빨라져.”

### Don’t (오행별 3개)
- wood(목): “시작도 안 했는데 완벽하려고 멈추지 말기.” / “새 일 벌리기 전에 ‘첫 5분’만.” / “비교하다가 기운 새지 말기.”
- fire(화): “감정 올라왔을 때 바로 보내기 금지(10분만 참기).” / “밤에 결제 버튼 누르지 말기.” / “말로 이기려고 하지 말기.”
- earth(토): “불안할수록 과한 정리/정돈에 도망치지 말기.” / “배고픈 상태로 결정하지 말기.” / “미래 걱정으로 현재 루틴 깨지 말기.”
- metal(금): “선 넘는 디테일 집착 금지(완료가 먼저).” / “상대 테스트하지 말기.” / “한 번에 다 끊으려다 폭발하지 말기.”
- water(수): “피곤한데 카페인으로 버티지 말기.” / “약속을 ‘의리’로만 잡지 말기.” / “기분 다운일 때 쇼핑으로 메우지 말기.”

### Lucky Item line (템플릿 5개)
- “{item} 하나만 ‘내 편’으로 두면, 오늘은 흔들려도 다시 돌아와.”
- “{item} = 오늘의 {lack} 부스터. 과열/불안이 내려가면 판단이 살아나.”
- “오늘은 {item}처럼 간단한 루틴이 제일 강한 부적.”
- “{item}은 ‘분위기 소품’이 아니라 ‘리듬 스위치’야.”
- “{item} 챙기면 오늘은 대충 해도 운이 덜 새.”

## UI 배치 제안(현재 결과 화면에 최소 변경으로 추가)

권장 순서
1) Result Card(새 섹션): headline/subhead/absurd line + share 버튼
2) 3 Signals(새 카드)
3) Buff/Debuff + Do/Don’t(새 카드 or 기존 insight-card 확장)
4) 메인 추천템(기존 유지, Lucky Item 라인 추가)
5) 추가 추천 리스트(기존 유지)
6) Gift Angle(옵션, CTA 근처에 배치)

## 공유(Share) 설계
### 복붙 캡션(ko) 3종
1) “나 오늘 {lack} 보강 뜸. {absurd_line} ㅋㅋ”
2) “{headline}… 이거 맞는 사람 있음? 결과: {main_item}”
3) “오늘의 부적템: {main_item}. 너는 뭐 뜸?”

### CTA 카피 리라이트(추천)
- 시작 CTA: “오늘의 부적템 받기”
- 공유 버튼: “친구 소환”
- 메인 아웃바운드: “오늘 운빨템 장착”
- 라이트/스트롱: “가볍게 보정” / “확실히 보정”

### 최소 이벤트(분석/성장)
- `start` (시작 CTA 클릭)
- `complete` (결과 화면 진입)
- `result_view` (결과 카드 노출)
- `share_click` (링크 복사/공유 버튼)
- `outbound_click` (쿠팡 링크 클릭: main/light/strong 구분)
